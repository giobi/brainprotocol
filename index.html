Memoria per Agenti AI

Dai al tuo agente
un cervello strutturato.

Una specifica aperta per costruire workspace di agenti AI leggibili dagli umani, scrivibili dalle macchine e versionati con git.

Markdown-first Leggibile dagli umani Versionato con git LLM-agnostico Nessun vendor lock-in

Non è un framework.
È una convenzione.

Brain Protocol definisce come la memoria di un agente AI debba essere organizzata su disco. Niente database, niente vector store, niente formati proprietari. Solo directory e file markdown.

Un brain è un repository git. Tutto ciò che l'agente sa — persone, progetti, task, cronologia — vive in markdown strutturato con metadati frontmatter. L'agente legge al boot, scrive durante le sessioni, committa il diff.

Solo testo

Ogni pezzo di memoria è un file markdown. Aprilo con qualsiasi editor, cercalo con grep, versionalo. Zero lock-in.

Git è il database

La memoria ha una storia. Fai diff tra due sessioni, ripristina una decisione, audita ogni modifica dell'agente.

Metadati frontmatter

Contesto machine-readable su ogni file: tipo, tag, relazioni, timestamp.

LLM-agnostico

Claude, Gemini, GPT, Mistral — qualsiasi modello che sa leggere una directory parla Brain Protocol.

La struttura delle directory.

Directory
wiki/ Entità: persone, aziende, progetti, tech, domini
diary/ Log temporale — cosa è successo e quando
todo/ Task aperti, tracciati in markdown strutturato
inbox/ Elementi in arrivo da processare
boot/ Configurazione di sistema: identità, agenti, tool, protocolli
tools/ Wrapper API, script di automazione, integrazioni
